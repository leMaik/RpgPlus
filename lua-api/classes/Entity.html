Methods

Entity:navigateTo (destination[, options][, callback])
Navigates to the given location, using A*.

Parameters:

  • destination destination, a table with x, y and z fields
  • options a table with options for pathing, options are openDoors (boolean), openFenceGates (boolean) and speed (int, in meters per second) (optional)
  • callback callback that is invoked when the entity has reached the destination (optional)

Returns:

    true if the navigation was possible, false if no path was found
Entity:teleportTo (destination)
Teleports the entity to the given location.

Parameters:

  • destination destination, a table with x, y and z fields and an optional world field (defaults to current world)
Entity:tell (player, message[, ...])
Tell a message to a player.

Parameters:

  • player player to tell the message to
  • message message to tell, either a string or a list of alternative string to randomly choose from
  • ... more messages to tell, each being either a string or a list of alternative string to randomly choose from (optional)
Entity:ask (player, question, callback)
Ask a player something. The player needs to send the answer using the chat.

Parameters:

  • player player to ask
  • question question, either a string or a list of alternative string to randomly choose from
  • callback callback that gets invoked with the response and a function to respond easily, respond(message) If the callback returns false, the question will be posed again
Entity:askChoices (player, question, choices, callback)
Ask a player a multiple choice question. The player can answer by either clicking on a choice or by writing the index of the choice in the chat.

Parameters:

  • player player to ask
  • question question, either a string or a list of alternative string to randomly choose from
  • choices choices that the user can select, each choice may be a string or a list of alternative strings to randomly choose from
  • callback callback that gets invoked with the response, the one-based response index and a function to respond easily, respond(message) If the callback returns false, the question will be posed again
Entity:startDialog (player, dialogDefinition)
Start a dialog with the given player.

Parameters:

  • player player
  • dialogDefinition a dialog definition

Respawn this entity. If this entity is already spawned, it will be despawned first.
Entity:on (event, handler)
Add a callback for the given event.

Parameters:

  • event

    event to add a handler to, possible events are:

    • "entity.click" - NPCClickRevent
    • "entity.click.right" - NPCClickRevent
    • "entity.click.left" - NPCClickRevent
    • "entity.addTrait" - NPCAddTraitEvent
    • "entity.collision" - NPCCollisionEvent
    • "entity.combust" - NPCCombustEvent
    • "entity.combust.byBlock" - NPCCombustByBlockEvent
    • "entity.combust.byEntity" - NPCCombustByEntityEvent
    • "entity.create" - NPCCreateEvent
    • "entity.damage" - NPCDamageEvent
    • "entity.damage.byBlock" - NPCDamageByBlockEvent
    • "entity.damage.byEntity" - NPCDamageByEntityEvent
    • "entity.death" - NPCDeathEvent
    • "entity.ender.teleport" - NPCEnderTeleportEvent
    • "entity.push" - NPCPushEvent
    • "entity.remove" - NPCRemoveEvent
    • "entity.removeTrait" - NPCRemoveTraitEvent
    • "entity.select" - NPCSelectEvent
    • "entity.spawn" - NPCSpawnEvent
    • "entity.attachTrait" - NPCTraitCommandAttachEvent
    • "entity.target" - EntityTargetNPCEvent
  • handler a handler function that is invoked whenever the event happens, gets the event as first parameter
Entity:once (event, handler)
Register an event handler that is only called once.

Parameters:

  • event event to add a one-time handler to
  • handler a handler function that is invoked whenever the event happens, gets the event as first parameter
Entity:off ([event[, handler]])
Remove an event handler.

Parameters:

  • event event to remove an event handler from; if not specified, all event handlers of this entity will be removed (optional)
  • handler the handler function to remove as previously added with on; if not specified, all handlers of this event will be removed (optional)

Returns:

    true if any event handler was removed, false if not
